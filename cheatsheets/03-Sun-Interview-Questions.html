üîç Technical Questions

Q: What is MVVM and why use it?
"Model-View-ViewModel separates concerns. Model holds data, View handles UI, ViewModel contains business logic and state.

Benefits: testable (test ViewModel without UI), maintainable (changes in one layer don't break others), and the View stays simple ‚Äî it just observes the ViewModel."
Q: How do you handle API errors?
"I check HTTP status codes first (200s = success, 400s = client error, 500s = server error). I parse error responses for user-friendly messages.

For the user, I show appropriate UI ‚Äî a retry button for network issues, clear message for validation errors. I also implement caching so users can continue using the app with stale data when offline."
Q: How do you optimise app performance?
"Profile first ‚Äî use Instruments (iOS) or Android Profiler. Common culprits:
‚Ä¢ Main thread blocking (move network/DB work off UI thread)
‚Ä¢ Memory leaks (use weak references, unsubscribe observers)
‚Ä¢ Unoptimised images (resize, cache, lazy load)
‚Ä¢ Unnecessary re-renders (check diffing, use proper state management)"
Q: How do you ensure code quality?
"Code reviews for every PR. Unit tests for business logic, UI tests for critical flows. Consistent coding standards (linting). CI/CD to catch issues early. And regular refactoring when code smells accumulate ‚Äî don't let tech debt compound."
