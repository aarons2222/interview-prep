ü§ñ Context & Memory

  • Activity context ‚Äî Dies with Activity. Use for UI.
  • Application context ‚Äî Lives forever. Use for singletons.
‚ö†Ô∏è Passing Activity context to singleton = memory leak

AsyncTask Leak

Inner class holds reference to outer Activity ‚Üí leak if Activity dies but task runs

Fix: static inner class + WeakReference

üçé ViewController Lifecycle

viewDidLoad ‚Üí viewWillAppear ‚Üí viewDidAppear ‚Üí viewWillDisappear ‚Üí viewDidDisappear
  • viewDidLoad ‚Äî Once, after view loaded. Setup.

  • willResignActive ‚Äî Losing focus
  • didEnterBackground ‚Äî Now in background
  • willEnterForeground ‚Äî Coming back
  • didBecomeActive ‚Äî Now active

üçé Memory Management (ARC)

Automatic Reference Counting ‚Äî compiler inserts retain/release

TypeBehavior
strongDefault. Keeps object alive.
weakOptional, becomes nil when dealloced
unownedNon-optional, crashes if dealloced

Retain Cycles

Two objects hold strong references ‚Üí neither deallocates

// ‚ùå Leak
closure { self.doThing() }

// ‚úÖ Fixed
closure { [weak self] in self?.doThing() }
