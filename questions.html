What happens when you rotate an Android device?

The Activity is destroyed and recreated. Full lifecycle: onPause ‚Üí onStop ‚Üí onDestroy ‚Üí onCreate ‚Üí onStart ‚Üí onResume.

To preserve data across rotation:

‚Ä¢ Use onSaveInstanceState() for small UI state

‚Ä¢ Use ViewModel ‚Äî survives configuration changes

‚Ä¢ Use android:configChanges in manifest (not recommended)

What's the difference between Serializable and Parcelable?

Serializable: Java standard, uses reflection, slower, easy to implement (just implement interface).

Parcelable: Android-specific, manual implementation, much faster, optimized for IPC.

Always say Parcelable for Android interviews ‚Äî it's the "correct" answer. 10x faster than Serializable.
Explain the Android Activity lifecycle

onCreate ‚Äî Activity created. Initialize UI, bind views, restore state.

onStart ‚Äî Becoming visible but not interactive yet.

onResume ‚Äî In foreground, user can interact. Start animations.

onPause ‚Äî Losing focus (dialog, partial cover). Pause animations, save drafts.

onStop ‚Äî No longer visible. Release heavy resources.

onDestroy ‚Äî Being destroyed. Final cleanup.

Mention onSaveInstanceState (called before onStop) for bonus points.
What is a ContentProvider and when would you use one?

A ContentProvider manages shared app data. It's the standard way to share data between applications.

Use cases:

‚Ä¢ Expose your app's data to other apps

‚Ä¢ Access system data (Contacts, MediaStore, Calendar)

‚Ä¢ Abstract data source from the rest of your app

Access via ContentResolver using URIs like content://com.example.provider/items

What's the difference between implicit and explicit intents?

Explicit: You specify the exact component (class name).

new Intent(this, DetailActivity.class)

Implicit: You declare an action, system finds a matching handler.

new Intent(Intent.ACTION_VIEW, Uri.parse("https://..."))

Explicit = internal navigation. Implicit = system features (camera, share, browser).
How do you handle memory leaks in Android?

Common causes:

‚Ä¢ Static references to Activity/Context

‚Ä¢ Inner classes (AsyncTask, Handler) holding Activity reference

‚Ä¢ Unregistered listeners (broadcast receivers, location)

Prevention:

‚Ä¢ Use Application context for singletons

‚Ä¢ Make inner classes static + WeakReference

‚Ä¢ Unregister in onStop/onDestroy

‚Ä¢ Use ViewModel instead of retained fragments

‚Ä¢ Use LeakCanary to detect leaks

Explain ARC and how it works

Automatic Reference Counting ‚Äî compiler inserts retain/release calls at compile time.

Each object has a reference count. When it hits zero, object is deallocated.

‚Ä¢ strong (default): Increments count, keeps object alive

‚Ä¢ weak: Doesn't increment, becomes nil when deallocated

‚Ä¢ unowned: Doesn't increment, crashes if accessed after dealloc

It's NOT garbage collection ‚Äî no runtime overhead, deterministic deallocation.
What's a retain cycle and how do you fix it?

A retain cycle happens when two objects hold strong references to each other ‚Äî neither can deallocate.

Common in closures:

self.closure = { self.doSomething() } ‚Äî self ‚Üí closure ‚Üí self

Fix with capture list:

{ [weak self] in self?.doSomething() }

Use weak when the captured object might become nil.

Use unowned when you're certain it will outlive the closure.

What's the difference between frame and bounds?

frame: Position and size in the superview's coordinate system. Where the view sits in its parent.

bounds: Position and size in the view's own coordinate system. Usually origin is (0,0).

Use frame to position a view in its parent.

Use bounds to draw content inside the view or scroll content.

Changing bounds.origin is how UIScrollView scrolls ‚Äî it shifts the internal coordinate system.
Explain the iOS app lifecycle

Not Running ‚Üí Inactive ‚Üí Active ‚Üí Background ‚Üí Suspended

Key callbacks:

‚Ä¢ didFinishLaunchingWithOptions ‚Äî App starting, one-time setup

‚Ä¢ applicationWillResignActive ‚Äî About to lose focus

‚Ä¢ applicationDidEnterBackground ‚Äî Save state, release resources

‚Ä¢ applicationWillEnterForeground ‚Äî Coming back, refresh UI

‚Ä¢ applicationDidBecomeActive ‚Äî Now active, restart tasks

What's the difference between struct and class in Swift?

Struct: Value type, copied on assignment, stack allocated, no inheritance.

Class: Reference type, shared on assignment, heap allocated, supports inheritance.

Apple recommends struct by default ‚Äî use class when you need:

‚Ä¢ Identity (=== comparison)

‚Ä¢ Inheritance

‚Ä¢ Shared mutable state

SwiftUI uses structs for views because they're value types ‚Äî safer, faster, no reference issues.
What are protocols and protocol extensions?

A protocol defines a contract ‚Äî required methods/properties without implementation.

A protocol extension provides default implementations ‚Äî adopters get functionality for free.

This enables "protocol-oriented programming" ‚Äî share behavior without inheritance.

Example: Equatable protocol + extension gives you != for free when you implement ==.

What is dependency injection and why use it?

Dependency Injection: Pass dependencies in from outside instead of creating them inside.

Instead of: class UserService { let api = APIClient() }

Do: class UserService { let api: APIClient; init(api: APIClient) }

Benefits:

‚Ä¢ Testability ‚Äî inject mock dependencies in tests

‚Ä¢ Flexibility ‚Äî swap implementations easily

‚Ä¢ Decoupling ‚Äî class doesn't know concrete types

Explain MVVM architecture

Model: Data and business logic

View: UI layer (Activity/ViewController/SwiftUI View)

ViewModel: Prepares data for display, handles UI logic, exposes observable state

The View observes the ViewModel. ViewModel doesn't know about the View.

Benefits over MVC:

‚Ä¢ Testable ‚Äî ViewModel has no UI dependencies

‚Ä¢ Separation ‚Äî View is dumb, ViewModel is smart

‚Ä¢ Reactive ‚Äî data binding keeps UI in sync

What's the difference between REST and GraphQL?

REST: Multiple endpoints, fixed data shape, over/under-fetching common.

GET /users/1, GET /users/1/posts

GraphQL: Single endpoint, client specifies exact data needed, one request.

query { user(id: 1) { name posts { title } } }

GraphQL = flexible but complex. REST = simple but rigid.

Most mobile apps use REST. GraphQL is growing but not universal yet.
How do you approach debugging a crash?

1. Read the stack trace ‚Äî find your code in the call stack

2. Reproduce ‚Äî can you make it crash again?

3. Check the obvious ‚Äî nil/null, index out of bounds, threading

4. Add logging ‚Äî narrow down where it fails

5. Use debugger ‚Äî breakpoints, step through

6. Check recent changes ‚Äî git diff, what changed?

Mention crash reporting tools: Firebase Crashlytics, Sentry, Bugsnag.
